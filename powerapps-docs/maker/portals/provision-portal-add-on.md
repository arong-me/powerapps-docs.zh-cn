# <a name="provision-a-portal-using-the-older-portal-add-on"></a>使用早期的门户加载项设置门户

如果您购买了早期的门户加载项，并且想要使用该加载项来设置门户，则必须转到 **Dynamics 365 管理中心**页面设置门户。

> [!NOTE]
> 若要设置门户，您必须被分派为门户选择的 Common Data Service 环境的系统管理员或系统定制员角色。 您还必须具有[所需权限](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#required-permissions)才能在 Azure AD 中创建和注册应用程序。 如果您没有所需权限，请与全局管理员联系来更新您的权限或要求全局管理员设置门户。

若要设置门户：

1. 登录到 [Microsoft 365 管理中心](https://admin.microsoft.com)。
 
2. 在左侧导航栏中，选择**全部显示**。

3. 如果在使用新管理中心，请选择**所有管理中心**。 从所有管理中心的列表中，选择 **Dynamics 365**。

    > [!div class="mx-imgBorder"]
    > ![从新管理中心打开 Dynamics 365 管理中心](media/from-new-admin-center.png "从新管理中心打开 Dynamics 365 管理中心") 

4. 如果在使用旧管理中心，请在左侧导航栏中展开**管理中心**，然后选择 **Dynamics 365**。

    > [!div class="mx-imgBorder"]
    > ![从旧管理中心打开 Dynamics 365 管理中心](media/from-old-admin-center.png "从旧管理中心打开 Dynamics 365 管理中心") 

5. 在 **Dynamics 365 管理中心**页，选择**应用程序**选项卡。

6. 选择标题为**门户加载项**的应用程序行，然后选择**管理**。

7. 在**常规设置**部分中，输入门户的**名称**。 **名称**可帮助标识门户，也可以在以后更改。

8. **类型**字段表示门户订阅的类型（“试用”或“生产”）。 这是系统字段，因此用户不能更改。 值将根据是试用订阅还是付费订阅更改。

9. 或者，在**门户开发状态**下拉列表中，为您的门户选择以下开发状态之一：

    - 原型
    - 开发
    - 测试
    - UAT
    - Live

    > [!NOTE]
    > - 对于现有的已设置门户，此下拉列表在**门户详细信息**选项卡上提供，默认情况下不选择状态。
    > - 此下拉列表仅对生产类型的门户可用。
    > - 此字段用于 Microsoft 了解此门户的使用模式，不会影响任何功能。 如果您为开发生命周期使用不同的名称，请选择更接近目的的名称。 此信息之后可以在门户设置后的某个时间点更改。

10. 在**门户 URL** 字段中，输入要用于您的门户的子域名称。 只能使用字母数字字符或连字符 (-)；不允许使用其他字符。

    > [!NOTE]
    > - 若要在设置门户后更改门户的 URL，请参阅[更改门户的基础 URL](admin/change-base-url.md)。
    > - 若要将您的门户链接到自定义域，请参阅[将门户链接到自定义域](admin/add-custom-domain.md)。

11. 在 **Dynamics 365 实例**下拉列表中，选择门户要链接到的实例。 这需要您所选实例中的系统管理员或系统定制员角色才能选择。

12. 在**选择门户语言**下拉列表中，选择您的门户的默认语言。 可用语言将取决于您的实例中安装的语言。 
    
    > [!NOTE]
    > 示例数据仅以一种语言提供，因此选择默认语言也将决定如何翻译示例数据。 阿拉伯语和希伯来语不受支持，因此不会在列表中显示。

13. 在**选择门户管理员**下拉列表中，选择将配置、自定义和维护门户的用户。 组织中所有拥有系统管理员角色的用户都将显示为选项。 

14. 在**门户受众**部分中，选择将访问新门户的受众的类型。 这将决定为您提供门户的哪些选项。 您可以选择：

    -   合作伙伴    
        -   客户自助服务门户
        -   自定义门户
        -   合作伙伴门户
        -   合作伙伴 Project Service（可选，需要安装解决方案）。
        -   合作伙伴 Field Service（可选，需要安装解决方案）
        -   社区门户

    -   客户
        -   客户自助服务门户
        -   自定义门户
        -   社区门户

    -   员工
        -   员工自助服务门户

15. 在**选择要部署的门户**部分中，选择要创建的门户类型。 您看到的选项基于您选择的受众。

    > [!div class="mx-imgBorder"]
    > ![配置门户的设置](media/configure-settings-portal.png "配置门户的设置")  

16. 选择**提交**并接受服务条款。
    > [!div class="mx-imgBorder"]
    > ![服务条款](media/terms-of-service.png "服务条款")  

在接受服务条款后，门户将开始设置。 设置通常需要 30 分钟，但也可能需要几个小时，具体取决于系统负载。 设置期间，“应用程序”选项卡上的门户*名称*将更改为 *Name*-Configuring。 导航回门户管理页面以检查设置是否成功。

在门户设置后，**门户详细信息**页将显示，其中包含所需的详细信息。

> [!div class="mx-imgBorder"]
> ![门户详细信息](media/portal-details-prov.png "门户详细信息") 


> [!Note]
> 在门户用户首次使用 Azure AD 凭据登录到门户时，将向所有用户（无论用户或门户类型）显示一个内容页。

## <a name="troubleshoot-provisioning"></a>故障诊断设置

有时包安装过程或 URL 创建过程可能出错。如果出错，可重新开始执行过程。

如果“*Name*-Configuring”更改为“*Name*-Provisioning 失败”，您需要重新开始执行设置过程。

1. 转至**应用程序**页面并选择门户。
2. 选择标签为**管理**的蓝色铅笔按钮。
3. 选择以下选项之一：

   - **重新开始设置**：使用以前定义的配置重新开始执行安装过程。

   - **更改值并重新开始设置**：重新开始执行设置过程前更改部分值。

如果包安装失败，将打开不含任何问题的“门户管理员”页面，但是导航到实际门户 URL 将显示消息“正在设置”。 若要确认此情况：

1. 转至 **Dynamics 365 管理中心**页面的“解决方案管理”页面并检查包状态是否为**安装失败**。 

2. 如果包状态为**安装失败**，请重新尝试从解决方案页面安装。 此外，请务必检查系统管理员在安装解决方案时是否将 Common Data Service 中的默认语言设置为应将门户安装为的语言。

> [!NOTE]
> 某些解决方案的安装需要满足一些先决条件，因此如果未满足这些先决条件，安装将失败。 例如，若要为合作伙伴门户安装合作伙伴 Field Service，必须已安装了合作伙伴门户和 Field Service 解决方案。 如果尝试先安装合作伙伴 Field Service，安装将失败并提供错误消息。
