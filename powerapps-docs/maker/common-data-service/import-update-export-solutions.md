# <a name="import-update-and-export-solutions"></a>导入、更新和导出解决方案 

 可以使用下列步骤手动导入解决方案。 仅导入从受信任来源获取的解决方案。 自定义项可能包括将数据发送到外部源的代码。 只能将默认解决方案导入到导出该解决方案的组织，而不能导入其他组织。  
  
1. 转到“[设置](../model-driven-apps/advanced-navigation.md#settings)” > “解决方案”。  
  
2.  在解决方案列表菜单中，选择“导入”。  
  
3.  在“导入解决方案”对话框中，执行“选择解决方案包”步骤以浏览到包含要导入的解决方案的压缩（.zip 或 .cab）文件。 
  
4.  选择“下一步”。  
  
5.  可以先查看解决方案的相关信息，再选择“导入”。  
  
6.  可能需要等待一段时间，解决方案导入才能完成。 如果导入成功，可以查看结果，并选择“关闭”。  
  
 如果已导入需要发布的任何更改，则必须发布自定义项后才可使用它们。 
  
 如果导入失败，则将看到一个报告，显示捕获的任何错误或警告。 可以选择“下载日志文件”，捕获有关导致导入失败原因的详细信息。 导致解决方案导入失败的最常见原因是，该解决方案未包含某些必需的解决方案组件。  
  
 下载日志文件时，你会发现一个 XML 文件，可使用 Office Excel 打开该文件并查看内容。  
  
> [!NOTE]
>  无法编辑活动传递规则集。 因此，如果要将包含活动传递规则集的解决方案导入到组织（其中已存在具有相同 ID 的规则）中，解决方案导入将失败。 详细信息：[创建自动传递案例的规则](https://docs.microsoft.com/dynamics365/customer-engagement/customer-service/create-rules-automatically-route-cases)

## <a name="update-solutions"></a>更新解决方案  
 有时可能希望向现有托管解决方案安装更新。 该过程类似于安装新托管解决方案，不同的是你会有一些不同的选项。 如果要更新从其他人处获取的解决方案，应从解决方案发布者处获取有关应选择哪些选项的指南。  
  
1. 转到“[设置](../model-driven-apps/advanced-navigation.md#settings)” > “解决方案”。   
  
2.  在解决方案列表菜单中，选择“导入”。  
  
3.  在“导入解决方案”对话框中，执行“选择解决方案包”步骤以浏览到包含要更新的解决方案的压缩（.zip 或 .cab）文件。  
4.  选择“下一步”。  
  
5.  可以先查看解决方案的相关信息，再选择“下一步”。 此页面将显示一个黄色栏，其内容为“此解决方案包包含已安装解决方案的更新”。  
  
6.  以下选项将可供选择：  
  
    - **保留自定义项(推荐)**  
  
         选择此选项将保留对组件执行的任何非托管自定义项，但也意味着此解决方案中包含的某些更新不会生效。  
  
    - **覆盖自定义项**  
  
         选择此选项会覆盖先前在此解决方案中所包含组件上执行的所有非托管自定义项。 此解决方案中包含的所有更新都会生效。  
  
     选择合适的选项，然后选择“下一步”。  
  
7.  可能需要等待一段时间，解决方案导入才能完成。 如果导入成功，可以查看结果，并选择“关闭”。  
  
 如果已导入需要发布的任何更改，则必须发布自定义项后才可使用它们。 
  
 解决方案发布者可能会要求导出现有非托管自定义项，使用覆盖自定义项的选项更新其托管解决方案，然后重新导入非托管自定义项。 这有助于确保应用预期的更改，同时预留自定义项。

## <a name="export-solutions"></a>导出解决方案  
 建议定期导出非托管自定义项，以便留有备份以防万一。 无法导出托管解决方案。  
  
1. 转到“[设置](../model-driven-apps/advanced-navigation.md#settings)” > “解决方案”。   
  
2.  在列表中选择想要导出的解决方案，然后选择“导出”。  
  
3.  在“发布自定义项”步骤中，系统会提醒仅会导出已发布的自定义项，而在选择“下一步”前，会提供选项让你选择“发布所有自定义项”。  
  
4.  如果解决方案缺少任何所需组件，则会看到“缺少所需组件”步骤。 仅当打算将其作为非托管解决方案导入回原始组织时，才可以忽略此警告。 否则，请按照对话框中的说明取消导出并添加必需组件。  
  
5.  在“导出系统设置(高级)”步骤中，可以选择要包含在解决方案中的某些系统设置。 如果解决方案依赖于任何系统设置的组，请选择这些组，然后选择“下一步”。  
  
     请参阅下面的“解决方案导出的设置选项”，了解有关每个选项将包含的设置详细信息。  
  
6.  在“包类型”步骤中，必须选择要将解决方案导出为“非托管”还是“托管”解决方案。  
  
7.  在下一步中，可选择用于特定 Dynamics 365 版本的目标解决方案。 可能希望导出符合先前版本的解决方案的 ISV 通常会使用此选项。 除非打算将此解决方案导入组织（其版本未升级到与你使用的组织版本相同），否则请接受默认值。   
  
8.  选择“导出”，下载解决方案文件。  
  
 下载文件的具体行为因浏览器而异。
