# <a name="overview-of-creating-apps-in-powerapps"></a>在 PowerApps 中创建应用程序概述
PowerApps 是一个高生产力的业务应用程序开发平台，它包含三个主要组件：

- [区域应用程序](canvas-apps/getting-started.md)从用户体验开始，借助空白区域设计出高度定制的界面，并将它连接到您选择的 200 个数据源。 您可以为 Web、移动和平板电脑应用程序构建区域应用程序。
- [模型驱动的应用程序](model-driven-apps/model-driven-app-overview.md)从数据模型开始 – 从 Common Data Service 中的核心业务数据和流程的模型进行构建来为表单、视图和其他组件建模。 模型驱动的应用程序会自动生成能够跨设备响应的绝佳 UI。
- [Common Data Service](common-data-service/data-platform-intro.md) 是 PowerApps 附带的一个数据平台，可让您存储业务数据并为其建模。 它是用于构建 Dynamics 365 应用程序的平台；如果您是 Dynamics 客户，则您的数据已位于 Common Data Service 中。

可以轻松地尝试构建您的第一个应用程序。 我们有一个 30 天的试用计划和一个免费的社区计划；找出最适合您的计划并开始使用。

## <a name="canvas-apps"></a>区域应用程序
区域应用程序使您能够按照所需方式灵活地安排用户体验和界面。 让您的创造力和商业意识来为您指明应用程序应有的外观。

您可以开始从数据所在的 Microsoft 工具构建您的应用程序，例如：

- [从 SharePoint 列表](canvas-apps/generate-app-from-sharepoint-list-interface.md)
- [从 Power BI 仪表板](canvas-apps/embed-powerapps-powerbi.md)

可以轻松创建区域应用程序；利用 PowerApps，您可以通过以下几种方式找到或创建您的应用程序：

- [从数据](canvas-apps/app-from-sharepoint.md)
- [从示例](canvas-apps/open-and-run-a-sample-app.md)
- [从 Common Data Service 源](canvas-apps/data-platform-create-app.md)
- [从空白区域](canvas-apps/data-platform-create-app-scratch.md)
- [通过 AppSource](../user/app-source.md)

## <a name="model-driven-apps"></a>模型驱动的应用程序
在创建模型驱动的应用程序时，可以借助 Common Data Service 快速配置表单、业务规则和流程。 从 PowerApps 站点创建模型驱动的应用程序。

开始使用模型驱动的应用程序是很简单的，您可以从以下主题开始：

- [创建应用程序](https://docs.microsoft.com/dynamics365/customer-engagement/customize/create-edit-app)
- [创建和设计窗体](https://docs.microsoft.com/dynamics365/customer-engagement/customize/create-design-forms)
- [创建或编辑视图](https://docs.microsoft.com/dynamics365/customer-engagement/customize/create-edit-views)
- [创建或编辑系统图表](https://docs.microsoft.com/dynamics365/customer-engagement/customize/create-edit-system-chart)
- [创建或编辑仪表板](https://docs.microsoft.com/dynamics365/customer-engagement/customize/create-edit-dashboards)
- [添加安全性](https://docs.microsoft.com/dynamics365/customer-engagement/customize/manage-access-apps-security-roles)
- [添加业务逻辑](https://docs.microsoft.com/dynamics365/customer-engagement/customize/guide-staff-through-common-tasks-processes)

## <a name="common-data-service-for-apps"></a>面向应用程序的 Common Data Service
Common Data Service 允许您在一组标准实体和自定义实体中安全地存储和管理数据，并且您可以在需要时向这些实体添加字段。

开始使用 Common Data Service 是很简单的。 例如，您可以从以下项目开始：
- [创建自定义实体](common-data-service/data-platform-create-entity.md)
- [管理字段](common-data-service/data-platform-manage-fields.md)
- [创建自定义选项集](common-data-service/custom-picklists.md)
- [创建业务规则](https://docs.microsoft.com/dynamics365/customer-engagement/customize/create-business-rules-recommendations-apply-logic-form)

在构建您的应用程序后，您可以与团队成员进行[共享](canvas-apps/share-app.md)。
