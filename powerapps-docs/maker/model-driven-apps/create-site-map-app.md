# <a name="tutorial-create-a-model-driven-app-site-map-for-an-app-using-the-site-map-designer"></a>教程：使用站点地图设计器为应用创建模型驱动应用站点地图

在本教程中，你将执行多个站点地图任务，例如创建新站点地图以及添加区域、组和子区域。

站点地图定义应用的导航。 使用基于图块的站点地图设计器轻松为应用创建站点地图。 使用设计器将组件拖到设计画布上，预览作品，并立即发布站点地图。 系统定制员或具有所需特权的任何用户都可以快速为应用创建站点地图。  
  
站点地图设计器也使你能够使用受环境支持的语言来定义区域、子区域或组标题。  
  
提供了一个默认的站点地图。 可以使用站点地图设计器编辑此站点地图或为新应用配置站点地图。 站点地图设计器与应用程序设计器相集成。  

## <a name="prerequisites"></a>先决条件
请确保具有系统管理员或系统定制员安全角色或等效权限。  具体而言，任何具有以下权限的用户也可以创建应用：  
-   应用实体的创建、读取和写入特权  
-   自定义实体的读写特权  
-   解决方案实体的读取特权

可以在安全角色的“自定义”选项卡上查看或设置这些特权。
  
## <a name="create-a-site-map-for-an-app"></a>为应用创建站点地图  
  
1. 在应用程序设计器画布上的“站点地图”区域中，选择“打开站点地图设计器”![“打开站点地图设计器”按钮](media/dynamics365-open-designer.PNG "“打开站点地图设计器”按钮")。  
  
     站点地图设计器将打开一个预先填充了一个区域、一个组和一个子区域的画布。 选择区域、组或子区域图块以更改其属性。  
  
    > [!NOTE]
    >  从应用程序设计器画布中选择“打开站点地图设计器”![“打开站点地图设计器”按钮](media/dynamics365-open-designer.PNG "“打开站点地图设计器”按钮")会自动创建新的站点地图（如果没有现有的站点地图），并赋予新站点地图与应用名称相同的名称以及与应用唯一名称相同的唯一名称。 

   ![选择站点地图](media/app-designer-sitemap-location.png "选择站点地图") 
  
2.  [向站点地图添加区域](create-site-map-app.md#bkmk_AddArea).  
  
3.  [向站点地图添加组](create-site-map-app.md#bkmk_AddGroup).  
  
4.  [在站点地图中向组添加子区域](create-site-map-app.md#bkmk_AddSubarea).  
  
5.  选择“保存”。  
  
    > [!NOTE]
    >  当返回应用程序设计器并选择“保存”时，新的站点地图将与应用相关联。 配置站点地图后，站点地图磁贴上将显示“已配置”；否则，图块上将显示“未配置”。  如果从应用程序设计器打开站点地图设计器并配置新站点地图，但在将站点地图与应用关联之前关闭浏览器，那么在下次打开应用程序设计器时，站点地图将根据应用唯一名称自动与应用关联。  
  
6.  选择“发布”。  
  
## <a name="edit-the-default-site-map"></a>编辑默认站点地图 

 环境将随附一个默认的站点地图。  
  
1. 打开解决方案资源管理器。  
  
2. 在解决方案窗口中的“组件”下，选择“客户端扩展”。  

3. 在组件工具栏上，选择“添加现有” > “站点地图”。

4. 在解决方案组件列表中，选择名为“站点地图”的站点地图，然后选择“确定”。
  
5.  双击以选择显示名称为“站点地图”且处于“托管”状态的所添加的站点地图。 也可以选择站点地图，然后在工具栏上选择“编辑”。  
  
     站点地图随即将在站点地图设计器中打开。  
  
6.  [向站点地图添加区域](create-site-map-app.md#bkmk_AddArea).  
  
7.  [向站点地图添加组](create-site-map-app.md#bkmk_AddGroup).  
  
8.  [在站点地图中向组添加子区域](create-site-map-app.md#bkmk_AddSubarea).  
  
9. 选择“保存”。  
  
10. 选择“发布”。  
  
<a name="bkmk_AddArea"></a>   
## <a name="add-an-area-to-the-site-map"></a>向站点地图添加区域  
  
1.  在站点地图设计器画布上选择“添加”![设计器上的“添加”按钮](media/dynamics365-designer-addbutton.PNG "设计器上的“添加”按钮")，然后选择“区域”。  
  
     或  
  
     将“区域”图块从“组件”选项卡中拖到画布上的空框中。 在将图块移动到画布上的正确位置时，你将看到此空框。  
  
2.  选择刚刚添加的区域。 你将在画布右侧窗格中看到突出显示的“属性”选项卡。  
  
3.  添加或编辑区域属性。  
  
     在“常规”下，执行以下操作：  
  
    - **标题**：以组织的基本语言输入区域的标题。  
  
    - **图标**：已选择了一个默认的应用程序图标。 从解决方案中可用的 Web 资源列表为此区域选择其他图标。  
  
    - **ID**：会自动生成一个唯一 ID，但可以按需输入其他 ID。 我们建议你使用提供的 ID，因为如果输入的 ID 不是唯一的，用户在使用应用时可能会遇到错误，或者当你导入包含此站点地图的解决方案时可能会出错。  
  
    - **显示组**：选中此复选框以在导航窗格中显示子区域组。  
  
     在“高级”下，执行以下操作：  
  
    - **更多标题**：如果组织使用多种语言，请为标题选择一种语言（区域设置），输入标题，然后选择“添加”![站点地图设计器中的“添加”按钮](media/add-icon-sitemap-designer.png "站点地图设计器中的“添加”按钮")。 可以为组织使用的所有语言创建、编辑或删除标题。 但是，每种语言只能有一个标题。  
  
    - **更多说明**：如果组织使用多种语言，请为说明选择一种语言，输入说明，然后选择“添加”![站点地图设计器中的“添加”按钮](media/add-icon-sitemap-designer.png "站点地图设计器中的“添加”按钮")。 可以为组织使用的所有语言创建、编辑或删除说明。 但是，每种语言只能有一个描述。  
  
    - **URL**：输入要为表示区域的 Dynamics 365 for Outlook 文件夹呈现的 URL。  
  
<a name="bkmk_AddGroup"></a>   
## <a name="add-a-group-to-the-site-map"></a>向站点地图添加组  
  
1.  在站点地图设计器画布上，选择要将组添加到的区域。  
2.  选择“添加”![设计器上的“添加”按钮](media/dynamics365-designer-addbutton.PNG "设计器上的“添加”按钮")，然后选择“组”。  
  
     或  
  
     将“组”图块从“组件”选项卡中拖到画布中“区域”下的空框中。 将图块移动到画布中的正确位置时，你将看到此空框。  
  
3.  选择刚刚添加的组。  
  
4.  在“属性”选项卡上，添加或编辑组属性：  
  
     在“常规”下，执行以下操作：  
  
    - **标题**：以组织的基本语言输入组的标题。  
  
    - **ID**：会自动生成一个唯一 ID。 如果需要，可以输入其他 ID。 我们建议你使用自动 ID，因为如果输入的 ID 不是唯一的，在导入包含此站点地图的解决方案时可能会出错。  
  
     在“高级”下，执行以下操作：  
  
    - **更多标题**：如果组织使用多种语言，请为标题选择一种语言（区域设置），输入组的标题，然后选择“添加”![站点地图设计器中的“添加”按钮](media/add-icon-sitemap-designer.png "站点地图设计器中的“添加”按钮")。 可以为组织使用的所有语言创建、编辑或删除标题。 但是，每种语言只能有一个标题。  
  
    - **更多说明**：如果组织使用多种语言，请为说明选择一种语言，输入组的说明，然后选择“添加”![站点地图设计器中的“添加”按钮](media/add-icon-sitemap-designer.png "站点地图设计器中的“添加”按钮")。 可以为组织使用的所有语言创建、编辑或删除说明。 但是，每种语言只能有一个描述。  
  
    - **URL**：输入要为表示组的 Dynamics 365 for Outlook 文件夹呈现的 URL。  
  
    - **设置为配置文件**：选中此复选框可指示此组是否代表适用于工作区的用户可选配置文件。 设置为用户可选配置文件的组在你的个人选项中作为选项提供。 这仅适用于“工作区”区域内的组。  
  
<a name="bkmk_AddSubarea"></a>   
## <a name="add-a-subarea-to-a-group-in-the-site-map"></a>向站点地图中的组添加子区域  
  
1.  在站点地图设计器画布上选择“添加”![设计器上的“添加”按钮](media/dynamics365-designer-addbutton.PNG "设计器上的“添加”按钮")，然后选择“子区域”。  
  
     或  
  
     将“子区域”图块从“组件”选项卡中拖到画布中“组”部分下的空框中。 将图块移动到画布中的正确位置时，你将看到此空框。  
  
2.  选择刚刚添加的子区域。  
  
3.  在“属性”选项卡上，添加或编辑子区域属性：  
  
     在“常规”下，执行以下操作：  
  
    - **类型**：选择要添加的子区域是仪表板、实体、Web 资源还是 URL。  
  
    - **实体**：选择子区域要用于的实体。 如果“类型”下拉列表中的子区域类型不是“实体”，则会禁用此字段。  
  
    - **URL**：指定在选中此子区域时要显示的应用程序主页面的 URL。 如果在“类型”下拉列表中选择了“实体”，则会禁用此字段。  
  
    - **默认仪表板**：选择要为此子区域显示的默认仪表板。 如果尚未在“类型”下拉列表中选择“仪表板”，则会禁用此字段。  
  
    - **标题**：以组织的基本语言输入子区域的标题。  
  
    - **图标**：已选择了一个默认的应用程序图标。 从解决方案中可用的 Web 资源列表为此子区域选择其他图标。  
  
    - **ID**。 会自动生成一个唯一 ID。 如果需要，可以输入其他唯一 ID。  
  
    - **参数传递**。 选中此复选框可将有关组织和语言上下文的信息传递给 URL。 仅当子区域类型为 Web 资源或基于 URL 的子区域时才可以选中此复选框。  
  
     在“高级”下，执行以下操作：  
 
    - **特权**：这定义了是否根据分配给用户的任何安全角色中可用的权限来显示子区域。 选择要检查特权的实体名称，然后选中复选框以分配特权。 
  
    - **更多标题**：如果组织使用多种语言，请为标题选择语言，输入子区域的标题，然后选择“添加”。 可以为组织使用的所有语言创建、编辑或删除标题。 但是，每种语言只能有一个标题。  
  
    - **更多说明**：如果组织使用多种语言，请为说明选择语言，输入子区域的说明，然后选择“添加”。 可以为组织使用的所有语言创建、编辑或删除说明。 但是，每种语言只能有一个描述。  
  
    - **SKU**：选择显示此子区域的 Dynamics 365 Customer Engagement 的版本。  
  
    - **客户端**：选择显示此子区域的客户端类型。  
  
    - **Outlook 快捷方式**：选择要在 Dynamics 365 for Outlook 中显示的图标。  
  
    - **脱机可用性**：选中此复选框，使用户在 Dynamics 365 for Outlook 中处于脱机状态时可以使用此子区域。  
  
## <a name="organize-areas-groups-and-subareas"></a>组织区域、组和子区域  
 可以通过将区域、组和子区域拖动到新位置来组织它们。 将出现一个可以在其中放置图块的容器框。 可以执行以下操作：  
  
-   将子区域移动到同一区域下同一组或不同组中的新位置。  
  
-   将子区域移动到不同区域下的组中的新位置。  
  
-   将组移动到同一区域内的新位置。  
  
-   将组移动到不同区域中的新位置。  
  
-   将区域移动到新位置。  
  
## <a name="clone-a-component-in-a-site-map"></a>在站点地图中克隆组件  
 若要制作现有组件的副本，请选择此组件，然后在工具栏上选择“克隆”。  除了 ID 和标题外，克隆组件的所有详细信息均与基本组件的信息相同。 ID 是随机生成的。 
  
 克隆区域时，克隆的区域将添加到当前所选区域的右侧。 克隆组时，克隆的组将添加到当前所选组的右侧。 克隆子区域时，克隆的子区域将添加到当前所选子区域的下方。  
  
## <a name="delete-an-area-group-or-subarea-from-a-site-map"></a>从站点地图中删除区域、组或子区域  
 若要删除站点地图组件，请选择组件图块，然后在工具栏上选择“删除”。 删除某个区域时，也会删除此区域中的所有组和子区域。 同样，删除某个组时，也会删除此组中的组和子区域。  
  
## <a name="clients-supported"></a>支持的客户端  
 下表说明了不同站点地图支持的客户端。  
 
|站点地图|支持的客户端|  
|---------------|-----------------------|  
|新应用| 统一接口和 Dynamics 365 Customer Engagement Web 应用 |  
|Dynamics 365 的站点地图 - 自定义应用 | Dynamics 365 Customer Engagement Web 应用和 Dynamics 365 for Outlook |  
|默认商业应用（销售、销售中心、客户服务、客户服务中心、现场服务、项目服务自动化）| Dynamics 365 Customer Engagement Web 应用和统一接口|
