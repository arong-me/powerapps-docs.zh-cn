# <a name="use-the-hospital-emergency-response-app"></a>使用医院应急响应应用

医院工作人员面临的挑战是，在紧急情况下在管理供应链的同时要应对患者人数的增加。 使用医院应急响应移动应用，一线工作人员可以快速查看和添加呼吸机、人员配备、等待出院患者以及与 2019 冠状毒病相关的患者的数据。

## <a name="prerequisites"></a>先决条件

若要开始使用此应用，您需要使用设备的应用商店在您的设备上下载 Power Apps Mobile。

- **下载** [**Power Apps Mobile**](https://powerapps.microsoft.com/downloads)
    - 对于装有 iOS 的 **Apple** 设备（例如 iPhone 和 iPad），请使用[**应用商店**](https://aka.ms/powerappsios)
    - 对于 **Android** 设备，请使用 [**Google Play**](https://aka.ms/powerappsandroid)
- 确保您的组织已按照[部署和配置应用](deploy-configure.md)中的说明部署和配置了医院应急响应应用。

安装 Power Apps Mobile 后，请从您的设备打开应用，然后使用公司的 Azure Active Directory 帐户登录。 登录后您可以查看共享到组织的所有应用。 有关详细信息，请参阅 [Power Apps 移动设备登录](https://docs.microsoft.com/powerapps/user/run-app-client#open-power-apps-and-sign-in)。

## <a name="hospital-emergency-response-app"></a>医院应急响应应用

![医院应急响应应用](media/use/app-launcher.png)

医院应急响应移动应用具有模块化结构，其中包含适用于您的角色的不同应用。 从 Power Apps Mobile 打开医院应急响应移动应用，选择您的**医院系统**、**区域，设施**，然后选择**下一步**开始使用。

> [!NOTE]
> *首次*启动医院应急响应移动应用或其任何组件时，系统会要求您同意以允许应用读取您的 *Office 365 用户*配置文件和*位置*。 必须选择**允许**，然后才能开始使用所选的应用。 有关详细信息，请参阅[授予同意](https://docs.microsoft.com/powerapps/user/run-app-client#give-consent)。

## <a name="app-components"></a>应用组件

![医院应急响应移动应用组件](media/use/app-components.png)

医院应急响应示例解决方案应用由多个应用组成，以增强用户体验。 根据您的角色，您可能会在**医院应急响应应用**中看到一个或多个组件。

- **人员 + 设备**
    <br> 按该设施中的位置收集 RN 和重要设备的状态。

- **用品**
    <br> 跟踪关键用品以更有效地跟踪、管理和预测库存。 

- **人员需求**
    <br> 按部门、角色和紧急性收集人员申请。

- **2019 冠状病毒统计**
    <br> 收集有关正在接受 2019 冠状病毒调查的患者人数以及测试呈阳性的人数的情况。

- **出院计划**
    <br> 收集患者出院的情况和预测。

## <a name="staff--equipment"></a>人员 + 设备

![人员 + 设备](media/use/staff-equipment.png)

为注册护士、患者和设备提交特定于位置的库存。 区域列表包含**医院应急响应应用**中所选设施的所有特定位置。 从可用选项中选择位置以更新其他字段。

选择区域后，为字段输入所需的值以将记录保存在解决方案数据库中。 您不需要在屏幕上为每个字段输入值。 为需要保存在解决方案数据库中的字段输入数字。

例如，如果您需要添加已申请的注册护士人数 3，在**当班注册护士 - 已申请**字段中输入 3，然后选择**发送**。 如果您还需要将使用中的呼吸机数更新为 6，在**当班注册护士 - 已申请**字段中输入 3，然后在**正在使用的设备**下的**呼吸机**中输入 6，然后选择**发送**。

如果要返回**医院应急响应应用**而不提交任何更改，请从左上角选择**返回**。 **提交**按钮用于提交您输入的值。

提交数据后，您可以选择返回到**人员 + 设备**应用，来使用**跟踪其他**按钮创建另一个记录。 选择**主页**返回**医院应急响应应用**。

### <a name="fields-and-description"></a>字段和说明

| **选项名称**               | **说明**                                                                                   |
|-------------------------------|---------------------------------------------------------------------------------------------------|
| 位置                      | 所选设施中房间、病房或任何其他专用位置的名称和类型。 |
| 患者人数            | 所选位置当前的患者总数。                                        |
| **当班的注册护士** |                                                                                                   |
| *合作伙伴*                    | 所选位置现存的注册护士搭档数。                             |
| *已请求*                   | 所选位置申请的注册护士数。                                  |
| *已分派*                    | 分派到所选位置的注册护士数。                                    |
| *未分派*                  | 未分派给所选位置任何任务的注册护士数。                    |
| **正在使用的设备**          |                                                                                                   |
| *呼吸机*                 | 所选位置正在使用的呼吸机数。                                            |
| *PAPR 罩*                  | 所选位置正在使用的送风过滤式呼吸器面罩数量。                 |
| *PAPR 带*                  | 所选位置正在使用的送风过滤式呼吸器带数量。                 |
| *PAPR 充电器*               | 所选位置正在使用的送风过滤式呼吸器充电器数量。              |


## <a name="supplies"></a>用品

![用品](media/use/supplies.png)

使用**用品**应用查看用品库存。 您可以通过此应用更新整个设施库存中的供应组件数量和每日消耗速度。

> [!NOTE]
> 在两个字段中输入值，**有存货**和**过去 24 小时使用量**，然后再选择**提交**。

如果要返回**医院应急响应应用**而不提交任何更改，请在左上角选择**返回**。 **提交**按钮用于提交您输入的值。 提交后，选择**主页**返回**医院应急响应应用**。

### <a name="fields-and-description"></a>字段和说明

用品应用项目列表可能会因您的组织要求而有所不同。 有关供应名称的说明，请参考您的组织资源。

IT 管理员可以使用 Power Apps 的模型驱动应用添加或更新用品应用项目列表。 有关详细信息，请参阅[配置指南](deploy-configure.md)。

> [!NOTE]
> 用品库存项目值必须采用数字格式。

## <a name="staffing-needs"></a>人员需求

![人员需求](media/use/staffing-needs.png)

收集所选设施的劳动力池请求。 确保已填写标记为*必填* (*) 的字段，然后才能提交设施的劳动力池请求。

如果要返回**医院应急响应应用**而不提交任何更改，请从左上角选择**返回**。 **提交**按钮用于提交您输入的值。 提交后，选择**主页**返回**医院应急响应应用**。

### <a name="fields-and-description"></a>字段和说明

| **字段名称**           | **说明**                                                                            |
|--------------------------|--------------------------------------------------------------------------------------------|
| 部门               | 提交劳动力请求的部门的名称。 此字段为*必填*字段。             |
| 部门位置      | 部门的位置。                                                                |
| 请求类型             | 劳动请求的类型，如“临床”和“非临床”。 此字段为*必填*字段。 |
| 所需角色              | 所申请的劳动力的角色，如保姆或注册护士。                          |
| 现在或下一个班次需要 | 为申请的人工、当前班次或下一个班次选择班次。                |
| 数量                 | 需要多少资源，采用数字格式。                |
| 详细信息                  | 描述有关劳动力池请求的其他详细信息或注释。                        |

## <a name="covid-19-stats"></a>2019 冠状病毒统计

![2019 冠状病毒统计](media/use/covid19-stats.png)

使用 **2019 冠状病毒统计**应用提交特定于 2019 冠状病毒的详细信息。您可以更新所调查患者以及发现呈阳性的患者的位置特定人数。

您还可以使用 **+ 添加另一个位置**按钮添加另一个位置，来发送多个位置的统计信息。

如果要返回**医院应急响应应用**而不提交任何更改，请从左上角选择**返回**。 **提交**按钮用于提交您输入的值。

提交数据后，您可以选择返回到 **2019 冠状病毒统计**应用，来使用**跟踪其他**按钮创建另一个记录。 选择**主页**返回**医院应急响应应用**。

### <a name="fields-and-description"></a>字段和说明

| **字段名称**  | **说明**                                                                                    |
|-----------------|----------------------------------------------------------------------------------------------------|
| 位置        | 所选设施中房间、病房或任何其他专用位置的名称和类型。  |
| PUI            | 正在接受调查的患者的人数。                                                            |
| 正数        | 2019 冠状病毒呈阳性的患者人数。                                                         |

## <a name="discharge-planning"></a>出院计划

![出院](media/use/discharge.png)

使用 **出院计划**应用提交出院信息和患者状况以及总数。 您可以更新过去 24 小时的出院详细信息，当前的出院障碍以及障碍的解除情况。

如果要返回**医院应急响应应用**而不提交任何更改，请从左上角选择**返回**。 **提交**按钮用于提交您输入的值。 提交后，选择**主页**返回**医院应急响应应用**。

### <a name="fields-and-description"></a>字段和说明

| **字段名称**            | **说明**                                                    |
|---------------------------|--------------------------------------------------------------------|
| 授权             | 处于授权流程的患者人数。                   |
| 耐用医疗设备 | 使用耐用医疗设备的患者人数。            |
| 监护              | 受监护的患者人数。                             |
| 家庭 + 社区服务 | 使用家庭或社区服务的患者人数。               |
| 安置                 | 需要安置的人数。                                       |
| 技术熟练的护理设施  | 技术熟练的护理设施数量。                              |
| **出院**            |                                                                    |
| 过去 24 小时                 | 预计在过去 24 小时内出院的患者人数。  |
| 未来 24 小时的可能人数          | 在过去 24 小时内出院的患者人数。                    |

## <a name="other-options"></a>其他选项

本节说明您可以使用医院应急响应移动应用组件执行的其他操作。

### <a name="end-shift---sign-out"></a>结束班次 - 注销

您可以使用屏幕左上角的个人资料图标从应用中注销。  

![注销](media/use/sign-out.png)

选择**结束班次**按钮结束您的会话并注销。

> [!NOTE]
> 如果您的 IT 管理员已禁用设备共享，*结束班次*可能不可用。

### <a name="provide-feedback"></a>提供反馈

您可以通过任何紧急响应移动应用组件中的**提供反馈**选项共享反馈。 若要共享您的反馈，请从左上角选择您的个人资料，然后选择**提供反馈**按钮：

![提供反馈](media/use/give-feedback.png)

选择**提供反馈**时，您可以使用应用选择共享称赞、想法或报告问题。

### <a name="switch-facility"></a>切换设施

![切换设施](media/use/switch-facility.png)

通过选择屏幕右上方的设施名称，可以随时切换设施。 选择位置名称后，您将转到**医院应急响应应用**屏幕，从中可以选择其他医院、区域或设施。
