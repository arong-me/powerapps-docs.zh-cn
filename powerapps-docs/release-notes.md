# <a name="whats-new-in-powerapps"></a>PowerApps 的最近更新
有关已知限制的信息，请参阅[常见问题和解决方法](common-issues-and-resolutions.md)。

**注意**：最新版 PowerApps 的版本号因平台而异。 为确保你使用的是适用于你平台的最新版 PowerApps，请从 Windows 应用商店、App Store 或 Google Play 下载安装。

## <a name="release-20700-20701"></a>版本 2.0.700-2.0.701
1. 在用户首次[保存应用](save-publish-app.md)后，默认每两分钟自动保存一次其他更改。
2. 可以轻松[创建规则](working-with-rules.md)来设置条件格式，而无需编写表达式。只需设置条件，然后直接在 PowerApps 画布上设计结果即可。
3. 使用添加控件时弹出的全高“数据”窗格，可以更轻松地配置窗体、库和数据表。
4. 可以获取有助于创建应用的上下文快速提示，无论是从空白应用、模板、数据源还是从 SharePoint 开始创建。

## <a name="release-20690-20691"></a>版本 2.0.690-2.0.691
1. 对于创建的应用，可以在 Power BI 嵌入式仪表板中[跟踪使用情况](app-analytics.md)。
2. 使用 [Split](functions/function-split.md) 函数，可以根据分隔符将文本字符串拆分成多个部分。

## <a name="release-20680-20681"></a>版本 2.0.680-2.0.681
1. 下载组织中使用的[应用列表](admin-view-apps.md)。
2. 为 SQL Server 委派 Sum、Average、Min 和 Max 函数。

## <a name="release-20670-20671"></a>版本 2.0.670-2.0.671
1. 在应用内显示 [Power BI 磁贴](controls/control-power-bi-tile.md)。
2. 使用 **[Coalesce](functions/function-isblank-isempty.md)** 函数可以更容易地替换空值，同时保留其他值不变。
3. 使用 **[EndsWith](functions/function-startswith.md)** 函数可以测试文本字符串是否以其他文本字符串结尾。
4. 在公式中使用 **[And](functions/operators.md)**、**[Or](functions/operators.md)** 或 **[Not](functions/operators.md)** 运算符。

## <a name="release-20660-20661"></a>版本 2.0.660-2.0.661
1. 使用右侧窗格可更容易地[配置控件属性](add-configure-controls.md)。
2. 设置输入控件的 DisplayMode 属性以支持显示数据和编辑数据，或两者皆不支持。
3. 使用 [Reset](functions/function-reset.md) 函数将输入控件重置为其默认值。
4. 使用 [Set](functions/function-set.md) 函数创建全局变量。
5. [在同一表单](functions/function-form.md)显示、创建或编辑记录。

## <a name="release-20650-20651"></a>版本 2.0.650-2.0.651
1. 保存对共享应用做出的更改，并[稍后发布](save-publish-app.md)。
2. 添加预填充库，其中包含丰富布局，并自动显示 Common Data Service 中一些最常用标准实体的数据。

4. 在[适用于 Web 的 PowerApps Studio](create-app-browser.md) 中，撤消/恢复更改和创建/编辑流。
5. 按名称指定默认未列出的 SharePoint 列表。
6. 使用改进后的 UI 配置“表单”控件。
7. 在应用打开时运行行为公式。

5. [同步编辑](edit-app.md)，以免损坏应用。
6. 在[单独环境](dev-community-plan.md)（始终免费）中培养技能。

## <a name="release-20600-20601"></a>版本 2.0.600-2.0.601
1. **实验性功能：使用实体窗体控件添加动态窗体，用户可以从 Common Data Service 中查看、导航和编辑其中的关系数据。**
   
    有关详细信息，请参阅[使用实体窗体控件](entity-form-control.md)。

3. **通过颜色编码的语法元素更轻松地读取长公式。**

4. **使用 PowerApps 通知连接从应用或流发送推送通知，并将任何其他应用作为目标。**
   
    选择应接收该通知的用户，并将消息和参数传递给目标应用，以获得完全自定义的体验。 有关详细信息，请参阅[向应用添加推送通知](https://powerapps.microsoft.com/blog/add-push-notification-to-you-app-and-boost-usage-and-retention/)。
5. **从管理中心重命名或删除环境。**
   
    有关详细信息，请参阅[环境管理](environments-administration.md)。
6. **通过指定 OpenAPI 定义 URL 来创建自定义连接器。**
   
    有关详细信息，请参阅[注册和使用自定义连接器](register-custom-api.md)。

## <a name="release-20590-20591"></a>版本 2.0.590-2.0.591
1. **显示调整大小以适应不同高度的项的库中的数据**。
   
    添加一个高度可变的库（如果有），例如，长度差异很大的文本（如产品说明）。 通过采用此方法，可以显示整个说明，而不会显示垂直库中的不必要空格。
   
    有关如何在库中显示数据的分步信息，请参阅[显示项的列表](add-gallery.md)。 有关参考信息，请参阅 [Gallery 控件](controls/control-gallery.md)。
2. **确定文本字符串是否以你指定的字符序列开头**。
   
    使用 [StartsWith 函数](functions/function-startswith.md)，以便用户可以执行以下操作，例如在搜索框中键入几个字母以基于字段开头的文本筛选项的列表。
3. **确定文本的字符串是否符合通过使用正则表达式指定的模式**。
   
    使用 [IsMatch 函数](functions/function-ismatch.md)基于字段是否在记录匹配项中返回一个 true/false 值，例如，姓名、电子邮件地址、电话号码或美国社会保障号。
4. **自定义当用户按 Tab 键时控件激活的顺序**。
   
    默认情况下，每个控件的 XY 坐标确定 Tab 键顺序。 若要指定不同的顺序，请将一个或多个控件的 **TabIndex** 属性设置为大于零的整数。 例如，如果某个控件的值设置为 1，则先激活该控件，然后再激活其值设置为 2 或更高值的控件。 通过添加工具提示并确保每个屏幕的 Tab 序列为最佳，可使应用[更易于访问](properties-accessiblity.md)。
5. **自定义滑块滑轨的粗细**。
   
    用户通过将控件图柄沿其滑轨拖动到右侧或左侧来调整[滑块控件](controls/control-slider.md)的值。
6. **在 Android 设备上，通过点击按钮而不是向下轻扫来刷新应用的列表**。
   
    有关详细信息，请参阅[在手机或平板电脑上使用 PowerApps](run-app-client.md)。
7. **Bug 修复和性能改进**。

## <a name="release-20531"></a>版本 2.0.531
1. **数种语言版本可用（含英语）**。

    选择某个控件时，其名称和任意父控件的名称显示在左下角附近的“面包屑”元素中。 双击右侧窗格顶部的控件名称或左侧导航栏中的屏幕名称可打开“**重命名**”文本框。
3. **可更轻松地查找工具栏上的选项**。

     查看和更新右侧窗格的“**高级**”选项卡中所选控件的属性值。

    自动生成应用时，默认浏览屏幕包含刷新按钮。 用户单击或点击此按钮即可在不关闭应用的情况下从数据源显示数据的最近更改。
6. **显示查找控件中的两个显示字段**。
7. **使用 Patch 函数可一次更新多个记录**。
   
    将基本记录表和更改记录表指定为 **[Patch](functions/function-patch.md)** 函数的参数即可一次创建或更新多个记录。
8. **可使用 ForAll 函数对一组记录执行计算和操作**。
   
    **[ForAll](functions/function-forall.md)** 函数对表的所有记录的公式求值。 该公式可以计算值和/或执行一个或多个操作，例如修改数据或使用连接。
9. **可使用 Power 函数计算指数值**。
   
    **[Power](functions/function-numericals.md)** 函数返回某数值的乘幂。 此函数相当于 [^ 运算符](functions/operators.md)。
10. **更好地跨会话和设备排序和筛选**。
    
     如果在移动设备上对应用列表进行排序或筛选，即使你关闭 PowerApps，该设置也会保留。 如果按最近打开应用的时间对应用列表进行排序，那么即使你在另一个设备上打开应用，排序顺序也是准确的。
11. **改进了机制以提供反馈**。
    
     通过单击或点击[我们的支持页](https://powerapps.microsoft.com/support/)上的“**联系人**”，然后提供有关问题信息即可创建票证，而不是单击或点击右上角附近的笑脸图标。

## <a name="release-20510-20512"></a>版本 2.0.510-2.0.512
因管理详细信息所致，不同的平台显示不同的内部版本号，但 2.0.510、2.0.511 和 2.0.512 已同时成功发布。

1. **在 iOS 10 中打开应用**。
2. **在 iOS 和 Android 设备上标记最喜爱的应用**。
   
    可以将应用标记为喜爱，并在可以运行的应用列表中仅显示最喜爱的应用。
3. **在 iOS 和 Android 设备上向下轻扫以刷新应用列表**。
4. **文本输入控件的延迟属性**。
   
   如果将 **[文本输入](controls/control-text-input.md)** 控件的 **[OnChange](https://powerapps.microsoft.com/en-us/tutorials/properties-core/)** 属性设置为某项操作，则默认情况下，每次用户在控件中键入一个字符均会执行该操作。 例如，用户可能在搜索框中键入了七个字母，那么每当用户添加一个字母都会更新结果，这会在搜索数据源时不必要地使用网络带宽。 如果将该控件的 **DelayOutput** 属性设置为 **true**，则用户键入一个或多个字符，然后在该控件中停止键入一段时间后，才会执行该操作。
5. **添加了数学函数**。
   
   * 计算正弦、余弦、正切及其他三角值。
   * 轻松访问 &pi; 的值。
   * 转换角度单位。
   * 计算自然指数和对数。
   
   有关详细信息，请参阅 [三角函数](functions/function-trig.md) 和 [数值函数](functions/function-numericals.md)。

   * 未使用 WADL 文件创建的 REST 服务
   * RSS
     
     如果打开应用进行编辑，出现的错误图标会告诉你哪个控件或哪些控件的属性引用了不推荐使用的数据源。

## <a name="release-20450"></a>版本 2.0.450
1. **在自定义窗体时添加数据**。
   
    选中窗体后，在右侧窗格中点击该窗体的数据源（或**未选定数据源**，然后点击“**添加数据源**”。
2. **现在完全委派从数据创建的应用来进行 SQL Azure 和 Salesforce 连接**。
   
   从数据创建应用来进行 SQL Azure 或 Salesforce 连接时，生成的应用现在会将所有操作都委派给服务。 现在可以使用生成的应用处理大型数据集。
   
   我们现在可以基于 **Search** 和 **SortByColumns** 函数生成公式，而不是基于 **Filter** 和 **Sort** 函数生成公式。  这些新函数不依赖于任何公式，因而可以更轻松地委派给数据服务。
3. **新 Search 函数**。
   
   **Search** 函数可以充当专用的 **Filter** 函数，使用单个字符串进行查找，并在文本列的列表内进行搜索。 如果搜索词不存在，此函数还将自动返回所有记录，适合在搜索**文本输入**控件中直接使用。 有关详细信息，请参阅 **Search** 函数文档，或查看从数据创建应用时生成的公式。
4. **运算符中现在支持委派**。
   
   可在 **Filter** 函数中随意使用此运算符来实现与 **Search** 函数相同的效果。
5. **显示受密码保护的 PDF 文件**。
   
    如果配置 **PDF 查看器**控件来显示需要密码的文件，则每个用户必须提供密码才能查看文件。 要删除密码要求，请将相应控件的**Password** 属性设置为相应文件的密码。
6. **DatePicker 控件中的协调世界时**。
   
    默认情况下，**DatePicker** 控件将解释本地时区中的所有日期值。 如果在协调世界时 (UTC) 中输入的时间是午夜，则它将在特定时区中显示前一天的日期。 例如，如果将 **DatePicker** 控件的 **DefaultDate** 属性设置为“2017-01-01T00:00:00Z”，则在太平洋时区 (UTC-0800) 中，则该控件将显示 2016 年 12 月 31 日。 若要确保控件正确显示这些值，请将其 **DateTimeZone** 属性从**本地**更改为 **UTC**。
